# 데이터 출처 (DATA_SOURCES)

> **단계적 하이브리드.** Phase 1 은 키리스 Open-Meteo 로 즉시 라이브, Phase 2 는 공공데이터포털 KHOA 실측 + 기상청 특보로 업그레이드한다. `DATA_PROVIDER=auto`는 두 키가 있으면 **`hybrid`**(KHOA 실측 우선 + 결측 지표 Open-Meteo 예보 백필 + KMA 특보), 없으면 `openmeteo`를 선택한다. 실키 검증(2026-07-11) 완료: 필드·단위·부이 센서 커버리지·특보 귀속 확정.

## 실측(KHOA) vs 모델(Open-Meteo)
- **Open-Meteo**: 세계 수치예보 모델(ECMWF·MeteoFrance·NOAA 계열)의 **예측값**. 키·가입 없이 좌표만으로 즉시 조회. 부산 연안 라이브 검증 완료. 비상업(공모전·데모) **무료**(공공누리 유사, 출처표시).
- **KHOA**: 실제 관측소·부표의 **실측값** → "지금 그 지점의 실제 상태" 신뢰도 최상. Phase 2 에서 채택 시 무환각 서사 강화.
- Open-Meteo 는 **기상특보를 제공하지 않음** → `supports_advisory=False`. 이때 특보는 '없음(결측 아님)'으로 처리하고 등급은 파고/풍속(임계 지표) 실측 기반으로 판단한다(RISK_THRESHOLDS.md).

## Phase 2 활용신청 대상 (확정)

### 1. KHOA 실측

- 상품명: **해양수산부 국립해양조사원_해양관측부이 최신 관측데이터**
- 공공데이터 ID / 신청: [`15155516`](https://www.data.go.kr/data/15155516/openapi.do)
- 제공기관: 해양수산부 국립해양조사원(해양예보과)
- 서비스 URL: `https://apis.data.go.kr/1192136/twRecent`
- 오퍼레이션: `GET /GetTWRecentApiService`
- 전체 요청 경로: `https://apis.data.go.kr/1192136/twRecent/GetTWRecentApiService`
- 주요 입력: 관측소 코드, 관측일자, 시간 간격, 공공데이터포털 인증키
- 주요 출력: 수온, 유향, 유속, 파고 등. 관측소별 센서 구성에 따라 일부 값은 없을 수 있다.
- 승인/한도: 개발계정 **자동승인**, 운영계정 **심의승인**; 개발계정 10,000건; 무료.

### 2. KMA 공식 특보

- 상품명: **기상청_기상특보 조회서비스**
- 공공데이터 ID / 신청: [`15000415`](https://www.data.go.kr/data/15000415/openapi.do)
- 제공기관: 기상청
- 서비스 URL: `https://apis.data.go.kr/1360000/WthrWrnInfoService`
- 사용 오퍼레이션: `GET /getPwnStatus` (특보현황조회)
- 전체 요청 경로: `https://apis.data.go.kr/1360000/WthrWrnInfoService/getPwnStatus`
- 주요 입력: `ServiceKey`, `pageNo`, `numOfRows`, `dataType`.
- 주요 출력: 현재 특보현황. 실제 응답의 구조화된 해역 필드는 발급 키로 확인해야 한다.
- 승인/한도: 개발·운영계정 모두 **자동승인**; 개발계정 10,000건; 무료.
- **주의:** 이 서비스는 특보 조회용이며 실측 `wind_speed`를 제공하지 않는다. 기존의 `wind_speed` 파싱은 제거했다. Phase 2에서도 풍속은 KHOA 관측값이 실제로 제공되는 관측소에 한해 사용하거나, Open-Meteo 풍속을 별도 유지해야 한다.

### 선정된 두 API의 한계와 선택적 세 번째 API

- 위 두 API만으로는 `tide_level`을 채울 수 없다. KHOA `twRecent`에는 조위가 명시되어 있지 않고 KMA 서비스는 특보 전용이다.
- 조위가 반드시 필요하면 세 번째 활용신청으로 **해양수산부 국립해양조사원_조위관측소 최신 관측데이터**([`15155508`](https://www.data.go.kr/data/15155508/openapi.do))를 추가한다.
- 선택 API 서비스 URL은 `https://apis.data.go.kr/1192136/dtRecent`, 오퍼레이션은 `GET /GetDTRecentApiService`이다. 조위·수온·유향·유속 등을 제공하지만 파고는 명시되어 있지 않다.

### 구형 바다누리 엔드포인트 폐기

- 이전 구현의 `http://www.khoa.go.kr/api/oceangrid/tideObsPreTab/search.do`는 구형 바다누리 링크 연계 방식이며 현재 코드에서 제거됐다.
- 공공데이터포털은 2026-01-12 기존 국립해양조사원 바다누리 연계 Open API 35종의 제공중단과 신규 국가중점데이터 사용을 [공식 공지](https://www.data.go.kr/bbs/ntc/selectNotice.do?originId=NOTICE_0000000004473)했다.
- 신규 호스트·경로·요청 형식과 방어 파서는 반영했지만, 실제 키 응답으로 필드·단위를 검증하기 전에는 Phase 2를 운영 검증 완료로 간주하지 않는다.

#### 부이별 센서 커버리지 (실키 검증 2026-07-11)

| 부이 | 관측소명 | 파고(wvhgt) | 풍속(wspd) | 유속(crsp) | 수온(wtem) |
|---|---|:---:|:---:|:---:|:---:|
| `TW_0062` | 해운대 | ✅ | ✅ | ✅ | ✅ |
| `TW_0090` | 송정 | ✅ | ✅ | ✅ | ✅ |
| `TW_0092` | 임랑 | ✅ | ✅ | ✅ | ✅ |
| `TW_0087` | 부산항 | ❌ | ❌ | ✅ | ✅ |

- **`TW_0087`(부산항)은 파고·풍속 센서가 없다 → 유속·수온만 제공.** 이 부이에 연결된 8개 지점(송도·다대포·부산항·감천항·다대포항·태종대·오륙도·이기대)은 파고·풍속 실측이 없다.
- **결정(2026-07-11, 사용자 명시 선택): 하이브리드 백필.** KHOA 실측이 없는 파고·풍속만 Open-Meteo 예보값으로 채운다(`HybridProvider`). 지표별 출처를 `metric_sources`(실측/예보)로 명시하며, UI·브리핑에서 구분 표기한다. KHOA가 제공하는 유속·수온은 실측을 유지하고 예보로 덮어쓰지 않는다. 이는 CLAUDE.md '결측 추정 금지' 원칙에 대한 **명시적으로 선택된 예외**이며, 은닉 대체가 아니다(출처가 항상 드러난다).
- 하이브리드에서도 특보는 KMA(공식)만 사용한다. 두 소스(KHOA+KMA) 모두 실패한 지점만 예보 파고·풍속으로 등급을 내고 특보는 '없음(결측 아님)'으로 둔다.

#### KMA 특보 귀속 (실키 검증 2026-07-11)
- `getPwnStatus` 응답의 현재 특보는 `t6` 필드에 `o <특보명> : <지역목록>` 형식으로 담긴다(`t7`=예비특보, 미반영).
- **육상 특보 오귀속 방지:** `t6`에 "부산"이 있어도 그것이 폭염/열대야 등 **육상 특보**일 수 있다(실측 확인됨). 각 특보 세그먼트의 지역 목록에 **부산 연안 앞바다 예보구역**(`남해동부앞바다` 등, `BUSAN_SEA_ZONES`)이 포함될 때만 풍랑/풍파 특보를 부산에 귀속한다. 나이브한 전체 blob 문자열 매칭(구버전)은 제주/먼바다 풍랑경보를 부산에 잘못 귀속하는 버그가 있었고 구조적 세그먼트 파서로 교체했다.
- 먼바다(안쪽/바깥먼바다) 특보는 연안 지점에 해당하지 않아 제외한다. 공식 해역 코드 필드가 확인되면 문자열 구역명 매칭을 코드 매칭으로 더 좁힐 수 있다.

## 수집 경로
- **주 경로:** `.github/workflows/snapshot.yml` 가 주기(3~6h) 실행 → 수집·정규화 → `backend/data/snapshot.json` 을 repo 에 commit-back. Render 무료티어 sleep·ephemeral FS 무관하게 내구성 확보.
- **폴백:** 수집 실패 시 마지막 성공 스냅샷 유지. 완전 오프라인 데모용 **seed 스냅샷**(라벨된 대표 데이터)을 커밋.
- **만료 판정:** 커밋 스냅샷 경로에서는 `snapshot_as_of`(수집 시각) 기준 `max_age` 판정.
