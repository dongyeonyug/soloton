# 위험도 임계값 (RISK_THRESHOLDS)

> ⚠️ **참고용 보수 매핑입니다.** 실제 기상특보만 공식 판단으로 취급하며, 파고·풍속 수치 비교는 특보 수치를 출발점으로 한 코드 규칙입니다. 어떤 인증·규제 기준도 아닙니다. `DISCLAIMER.md` 참조.

위험도 엔진(`backend/app/engine/`)은 **결정론적 순수함수**입니다. 시계·난수를 쓰지 않으며, 동일 입력은 항상 동일 등급을 냅니다. 모든 임계값은 코드에서 `source` 문자열을 동반합니다(사실의 원천).

## 등급 정의
- `SAFE` (안전) · `CAUTION` (주의) · `DANGER` (위험)
- 최종 등급 = 파고·풍속 수치 매핑의 **worst**. 활성 기상특보는 자신의 등급 이상으로 override.

## 지표별 임계값

| 지표 | SAFE | CAUTION | DANGER | 코드 규칙의 출발점 | 근거 성격 |
|------|------|---------|--------|--------------|-----------|
| 풍속 (m/s) | < 14 | 14 ~ 21 | ≥ 21 | 기상청 풍랑특보의 풍속 수치(14/21m/s)를 단일 시점 비교용으로 보수 적용 | **보수 매핑** |
| 유의파고 (m) | < 2 | 2 ~ 3 | ≥ 3 | 기상청 풍랑특보 파고 수치(3/5m)를 연안 참고용으로 더 낮게 적용 | **보수 매핑** |
| 조류 (m/s) | - | - | - | 지역·활동별 검증 임계값 부재 | **참고값, 등급 비반영** |

### 근거 성격의 의미
- **공식 기준:** 기상청이 실제로 발표한 특보다. 아래 특보 override에서만 최종 등급을 직접 올린다.
- **보수 매핑:** 기상청 특보 수치를 출발점으로, 한 번의 관측·예보값을 비교하는 서비스용 코드 규칙이다. 기상청 풍랑특보는 풍속 또는 유의파고가 기준 이상으로 **3시간 이상 지속되거나 예상될 때** 발표되므로, 이 서비스의 단일 시점 수치 등급은 특보 판정 자체가 아니다. [기상청 풍랑 특보 발표 기준](https://www.weather.go.kr/kma/biz/forecast03.jsp)
- **참고값, 등급 비반영:** 관측값은 출처·시각과 함께 표시하지만, 검증된 지역 기준이 없어 최종 `SAFE`/`CAUTION`/`DANGER`에는 사용하지 않는다.

## 기상특보 override

| 특보 | 최소 강제 등급 |
|------|----------------|
| 풍랑주의보 / 풍파주의보 | CAUTION |
| 풍랑경보 / 풍파경보 | DANGER |

## T11 공식 특보 재현 검증

`backend/data/advisory_replay_cases.json`에는 기상청 **기상특보목록**의 실제 발표 레코드 두 건을 보관한다. 테스트는 그 특보 종류를 엔진에 넣었을 때, 수치 입력이 낮아도 풍랑주의보는 최소 `CAUTION`, 풍랑경보는 최소 `DANGER`가 되는지 검증한다.

- 출처: [기상청 기상특보 조회서비스 (`15000415`)](https://www.data.go.kr/data/15000415/openapi.do)의 `getWthrWrnList`. 이 기능은 발표 시각 범위와 지점 코드로 특보 목록을 조회한다.
- 범위: 레코드는 특보 override 규칙만 검증한다. 케이스의 파고·풍속은 override를 고립하기 위한 **합성 제어 입력**이며, 해당 시점의 현장 관측값·사고·이용자 안전 결과를 재현하는 데이터가 아니다.
- 보존 한계: 2026-07-16 실제 조회에서 API가 오늘 기준 6일보다 이전 기간을 거부했다. 따라서 이 파일은 장기 과거 특보 아카이브나 규칙의 정확도 통계가 아니다.
- 실행: `cd backend && ./.venv/bin/pytest -q tests/test_advisory_replay.py`

## 단일 해안 활동 참고 기준
활동별 임계값 보정은 근거가 검증되지 않아 사용하지 않는다. 파고·풍속만 위 공통 매핑으로 평가하며, 조류·조위·수온은 참고값이다. 이 등급은 특정 활동의 허가·출항·입수 판단이 아닌 참고용 요약이다.

## 위험도 계산 경로
각 브리핑은 코드가 만든 `decision_steps`를 함께 반환한다. 파고·풍속의 중간 등급, 조류의 등급 비반영 사유, 특보의 상향 여부, 결측으로 인한 최소 주의 처리, 최종 합산 순서가 포함된다. AI 산문은 이 경로를 수정하거나 새로운 판단 근거를 만들 수 없다.

예보 기반 시간대에는 `safe_window`에 선택 규칙, 평가 지평(기본 12시간), 전체 후보 시각 수, 실제 등급을 낸 시각 수, 선택 구간의 시각 수를 기록한다. SAFE 시각을 우선하고, 없으면 CAUTION 시각 중 가장 이른 연속 구간을 고른다. DANGER만 있거나 예보가 불완전하면 시간대를 만들지 않는다.

## 결측 → 등급 규칙 (안전-치명적)
임계 지표(파고/풍속/특보) 중 하나라도 `is_missing` 이면:
- 등급은 **절대 SAFE 불가**, 최소 **CAUTION floor**
- `has_missing_critical = true`
- 브리핑은 해당 지표를 '정보없음'으로 자백(`is_confession = true`)

## 신선도 만료
지표별 `observed_at` + `max_age` 초과 시 `is_missing` 으로 강등. 예: 특보 3h, 파고/풍속/조류 6h. 커밋 스냅샷 경로에서는 `snapshot_as_of` 기준으로 판정.

## 데이터 상태 표기
브리핑은 지표마다 다음을 함께 표시한다.
- **실측 / 예보 / 값 확인됨:** 수집된 값의 출처 유형. 수치와 관측 시각을 병기한다.
- **정보없음:** 수집 결과에 값이 없어 추정하지 않는다. 값의 출처가 아니라, 확인을 시도한 출처를 따로 표시한다.
- **시간 경과:** 값은 있었지만 신선도 기준을 넘어 위험도 입력에서 제외했다.
- **핵심값:** 파고·풍속·특보. 하나라도 없으면 안전으로 등급을 내리지 않고 `CAUTION` 이상으로 보수 처리한다.
- **참고값:** 조류·조위·수온. 화면에는 보이지만 현재 위험도 등급에는 반영하지 않는다. 조류는 지역·활동별 검증 임계값을 확보하기 전까지 이 상태를 유지한다.

## 특보 소스 부재 처리 (provider별)
기상특보를 제공하는 provider(KHOA/기상청)와 그렇지 않은 provider(Open-Meteo)를 구분한다.
- **특보 미지원 provider(Open-Meteo):** 특보는 '없음(NONE, 결측 아님)'으로 둔다. 등급은 **파고·풍속 실측(예측) 기반**으로만 판단하며, 특보 부재를 이유로 전 지점을 CAUTION 으로 강등하지 않는다.
- **결측-플로어는 여전히 파고/풍속(임계 지표)에만 적용:** 이 둘 중 하나라도 결측이면 안전 불가 + CAUTION floor(무환각 보수화 유지).
- **특보 지원 provider(Phase 2):** 실제 특보를 수집해 override 로직이 정상 복귀한다.
